## 왜 단축키인가

Claude Code는 프롬프트만 잘 쓰면 되는 줄 알았다. 근데 실제로 며칠 쓰다 보면 프롬프트 쓰는 시간보다 **프롬프트를 수정하고, 결과를 확인하고, 되돌리는 시간**이 더 길다. 단축키 몇 개 알아두면 그 삽질 시간이 확 줄어든다.

> **참고:** 한글 입력 모드(IME)가 켜져 있으면 `Ctrl` 조합이 안 먹힌다. 단축키 쓸 때는 영문 모드로 전환하고 누르자.

## 단축키 정리

1. **`Ctrl+Z` — 백그라운드 전환**

   Claude를 백그라운드로 보내고, `fg`로 복귀 가능.

   긴 작업 중 터미널에서 뭔가 확인할 때 유용.

2. **`!` — Bash 모드**

   입력창에 `!`치면 바로 셸 명령 실행 가능.

   Claude한테 시키면 도구 호출 오버헤드 발생하니 직접 호출이 성능상 유용.

3. **`Ctrl+G` — 외부 에디터**

   기본 에디터(vim, nano 등)에서 프롬프트를 여러 줄로 편하게 작성.

   복잡한 요구사항 정리할 때 유용.

4. **`Ctrl+S` — 프롬프트 스태시**

   쓰던 프롬프트를 임시 저장하고 다른 명령 실행.

   다시 `Ctrl+S`로 복구. Git의 `stash`랑 같은 개념.

5. **`Esc` + `Esc` — 롤백**

   코드와 대화를 이전 체크포인트로 동시에 되돌린다.

   Claude가 이상한 방향으로 갔을 때 유용.

6. **`Ctrl+W` — 단어 삭제**

   커서 앞 단어를 통째로 날린다.

   Backspace 연타 대신 사용.

7. **`Ctrl+A` / `Ctrl+E` — 줄 처음/끝 이동**

   긴 프롬프트에서 화살표 연타 대신 한 방에 이동.

   터미널 기본 단축키라 어디서든 동작.

8. **`Shift+Tab` — 권한 모드 전환**

   Normal → Auto-Accept → Plan 모드를 한 키로 전환.

9. **`Ctrl+L` — 화면 클리어**

   화면만 밀고 대화 히스토리는 유지.

   출력이 길어져서 지저분할 때 사용.

10. **`Ctrl+R` — 히스토리 검색**

    이전에 입력했던 프롬프트를 키워드로 찾아서 재사용.

    비슷한 요청 반복할 때 타이핑 안 해도 된다.

11. **`@` — 파일 경로 자동완성**

    `@src/comp...` 식으로 치면 프로젝트 파일 경로를 제안해준다.

    오타 걱정 없이 정확한 파일 지정.

12. **`/compact` — 컨텍스트 압축**

    대화가 길어지면 `/compact`로 요약해서 토큰을 줄인다.

    `/compact 타입 관련 내용 유지해줘` 같이 유지할 내용도 지정 가능.
