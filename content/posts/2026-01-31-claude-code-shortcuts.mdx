## 왜 단축키인가

Claude Code는 프롬프트만 잘 쓰면 되는 줄 알았다. 근데 실제로 며칠 쓰다 보면 프롬프트 쓰는 시간보다 **프롬프트를 수정하고, 결과를 확인하고, 되돌리는 시간**이 더 길다. 단축키 몇 개 알아두면 그 삽질 시간이 확 줄어든다.

## 단축키 정리

### `Ctrl+Z` — 잠깐 멈추고 터미널 쓰기

Claude가 긴 작업을 돌리고 있는데 다른 터미널에서 뭔가 확인해야 할 때. `Ctrl+Z`로 Claude를 백그라운드로 보내고, 볼 거 보고, `fg`로 복귀하면 된다. 별도 터미널 탭을 안 열어도 된다는 게 핵심이다.

### `!` — Bash 모드 전환

입력창에 `!`를 치면 바로 터미널 명령어를 실행할 수 있다. `git status`나 `ls` 같은 간단한 명령을 Claude한테 시키면 도구 호출이 돌아가면서 오버헤드가 생기는데, `!`로 직접 치면 즉시 결과가 나온다. Claude한테 시키기엔 애매한 간단한 명령은 이걸로 처리하면 된다.

### `Ctrl+G` — 외부 에디터로 프롬프트 편집

긴 프롬프트를 한 줄짜리 입력창에서 쓰려면 답답하다. `Ctrl+G`를 누르면 기본 에디터(vim, nano 등)가 열리고, 거기서 여러 줄에 걸쳐 편하게 작성한 뒤 저장하면 프롬프트로 들어간다. 복잡한 요구사항을 정리할 때 필수다.

### `Ctrl+S` — 프롬프트 임시 저장(Stash)

프롬프트를 반쯤 쓰다가 "잠깐, 현재 상태부터 확인해야겠다" 싶을 때. `Ctrl+S`로 쓰던 내용을 치워두고, 다른 명령을 실행한 뒤, 다시 `Ctrl+S`를 누르면 아까 쓰던 프롬프트가 복구된다. Git의 `stash`랑 같은 개념이다.

### `Esc` + `Esc` — 코드 + 대화 함께 롤백

Claude가 이상한 방향으로 가버렸을 때 최고의 탈출구다. `Esc`를 두 번 연타하면 이전 체크포인트로 코드와 대화를 동시에 되돌린다. Git commit 없이도 rewind가 가능하다는 게 핵심이다. "아, 이거 아닌데" 싶으면 바로 눌러주면 된다.

### `Ctrl+W` — 단어 단위 삭제

Backspace 연타하면서 한 글자씩 지우는 건 비효율의 극치다. `Ctrl+W`는 커서 앞의 단어를 통째로 날린다. 파일 경로나 긴 옵션명을 잘못 썼을 때 특히 유용하다.

### `Ctrl+A` / `Ctrl+E` — 줄 처음/끝 점프

긴 프롬프트의 앞부분을 수정하려고 화살표 키를 꾹 누르고 있었다면, `Ctrl+A`로 줄 처음으로, `Ctrl+E`로 줄 끝으로 바로 이동할 수 있다. 터미널 기본 단축키인데 Claude Code에서도 그대로 동작한다.

### `Shift+Tab` — 권한 모드 전환

Normal 모드, Auto-Accept 모드, Plan 모드를 `Shift+Tab` 하나로 전환할 수 있다. Auto-Accept는 Claude가 파일 수정이나 명령 실행을 물어보지 않고 바로 진행하는 모드인데, 신뢰할 수 있는 작업에서 켜두면 승인 클릭 없이 쭉 진행되니까 속도가 확 다르다. 위험한 작업 전에는 다시 Normal로 돌려놓으면 된다.

### `Ctrl+L` — 화면 클리어

터미널의 `clear`와 같다. 화면만 깨끗하게 밀어주고 대화 히스토리는 그대로 유지된다. 출력이 길어져서 화면이 지저분할 때 쓰면 된다.

### `Ctrl+R` — 히스토리 검색

이전에 입력했던 프롬프트를 다시 쓰고 싶을 때. `Ctrl+R`을 누르고 키워드를 치면 매칭되는 이전 입력을 찾아준다. 비슷한 요청을 반복할 때 처음부터 다시 타이핑할 필요가 없다.

### `@` — 파일 경로 자동완성

프롬프트에서 `@`를 치면 프로젝트 파일 경로를 자동완성해준다. "이 파일을 수정해줘"라고 할 때 경로를 직접 타이핑하는 대신 `@src/comp...` 식으로 치면 자동으로 제안이 뜬다. 오타 걱정 없이 정확한 파일을 지정할 수 있다.

### `/compact` — 컨텍스트 압축

대화가 길어지면 토큰을 많이 잡아먹으면서 Claude 응답이 느려지거나 비용이 올라간다. `/compact`를 치면 지금까지의 대화를 요약해서 컨텍스트를 줄여준다. `/compact 타입 관련 내용 유지해줘` 같이 유지할 내용을 지정할 수도 있다.
